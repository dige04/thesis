## Implementation Details

### 1. Core Components

#### CostTracker Class
Main class for tracking costs throughout a run:
- Tracks individual LLM and embedding calls
- Aggregates costs per task
- Aggregates costs per run
- Writes cost summaries to JSON
- Supports budget monitoring

**Key Methods:**
```python
tracker = CostTracker(run_id="run_001", run_dir="runs/run_001")
tracker.start_run(policy="type_aware_decay", sequence_name="django", seed=1)
tracker.track_llm_call(call_type="agent", model="gpt-4o-mini", ...)
tracker.track_embedding_call(model="text-embedding-3-small", ...)
tracker.complete_task(task_id)
tracker.complete_run()
tracker.write_cost_summary()
```

#### Data Classes

**LLMCallCost:**
- Tracks single LLM call (agent, classifier, consolidation)
- Records prompt/completion tokens
- Computes estimated cost in USD
- Stores metadata (temperature, max_tokens, etc.)

**EmbeddingCallCost:**
- Tracks single embedding call
- Records token count
- Computes estimated cost in USD
- Links to task_id and memory_id

**TaskCostSummary:**
- Aggregates costs for a single task
- Breaks down by call type (agent, classifier, consolidation, embedding)
- Tracks call counts and token usage

**RunCostSummary:**
- Aggregates costs for entire run (sequence)
- Includes all task summaries
- Tracks run metadata (policy, sequence, seed)
- Records start/end timestamps

**Cost Calculation:**
```python
input_cost = (prompt_tokens / 1_000_000) * pricing["input"]
output_cost = (completion_tokens / 1_000_000) * pricing["output"]
total_cost = input_cost + output_cost
```

### 3. Cost Aggregation Levels

#### Per-Call Level
Every LLM and embedding call is logged with:
- Unique call ID
- Call type (agent, classifier, consolidation)
- Model name
- Token counts
- Estimated cost
- Timestamp
- Associated task_id
- Metadata

#### Per-Task Level
Aggregates all calls for a task:
- Agent LLM cost and call count
- Classifier cost and call count
- Consolidation cost and call count
- Embedding cost and call count
- Total cost
- Total tokens by type

#### Per-Run Level
Aggregates all tasks in a sequence:
- Total cost across all tasks
- Breakdown by cost type
- Total LLM and embedding calls
- Total tokens used
- List of task cost summaries
- Run metadata (policy, sequence, seed)

#### Cross-Run Level
Daily cost reports aggregate across all runs:
- Total cost across all runs
- Cost breakdown by policy
- Cost breakdown by date
- List of all run summaries

### 5. SequenceRunner Integration
Aggregate and write cost summary:
```python
class SequenceRunner:
    def __init__(self, run_id, policy, config):
        # Initialize cost tracker
        self.cost_tracker = CostTracker(
            run_id=run_id,
            run_dir=self.run_dir
        )
        
    def run_sequence(self, sequence, seed):
        # Start cost tracking
        self.cost_tracker.start_run(
            policy=self.policy.name,
            sequence_name=sequence.sequence_name,
            seed=seed
        )
        
        for task in sequence.tasks:
            self.cost_tracker.start_task(task.task_id)
            task_result = self._execute_task(task, seed)
            task_cost = self.cost_tracker.complete_task(task.task_id)
            
            # Update task result with costs
            task_result.task_api_cost = task_cost.agent_llm_cost
            task_result.consolidation_llm_cost = task_cost.consolidation_cost
            
        # Complete run and write summary
        run_summary = self.cost_tracker.complete_run()
        self.cost_tracker.write_cost_summary()
        
        return SequenceResult(
            total_cost_usd=run_summary.total_cost,
            ...
        )
```

## Notes

### Pricing Updates
The pricing constants in `PRICING` dictionary should be updated if OpenAI changes their pricing. Current pricing is accurate as of implementation date.

### Cost Estimation
Costs are **estimated** based on token counts and published pricing. Actual costs may vary slightly due to:
- API pricing changes
- Rounding in token counting
- Additional API fees (if any)

For accurate cost tracking, compare cost summaries with actual OpenAI billing.

### Budget Monitoring
Budget alerts are **informational only** and do not prevent execution. To enforce hard budget limits, integrate with OpenAI's usage limits API or implement execution gates in the experiment runner.

### Performance
Cost tracking adds minimal overhead:
- In-memory tracking (no I/O per call)
- JSON writes only at task/run completion
- Daily reports generated on-demand
